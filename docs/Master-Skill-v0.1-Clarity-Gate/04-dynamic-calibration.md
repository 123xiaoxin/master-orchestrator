# Dynamic Calibration

Clarification is not a fixed pre-execution ritual. It is an ongoing calibration mechanism.

## When To Pause

Master should pause execution when:

- the observed task no longer matches the real goal
- a critical variable is missing
- risk rises beyond the current contract
- user assumptions conflict with implementation reality
- the minimum prototype reveals direction mismatch
- an execution layer reports ambiguity that changes scope

## How To Calibrate

When calibration is needed:

1. Stop expanding execution.
2. State the detected mismatch or missing variable.
3. Preserve what has already been verified.
4. Ask the smallest necessary question or present a revised prototype.
5. Update the execution contract before resuming.

## What Calibration Is Not

Calibration is not restarting from scratch. It is not a long interview. It is not permission to add Agents or process by default.

Calibration keeps the system aligned while minimizing interruption.

## Execution Contract Update

Any material change discovered during execution should update the contract fields it affects:

- real goal
- boundaries
- allowed actions
- forbidden actions
- minimum prototype
- validation metrics
- risk boundaries
