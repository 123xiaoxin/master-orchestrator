# 🔄 MasterOrchestrator — Quality 质量飞轮

> **版本**：v1.1 | **定位**：系列插件，需搭配 Core 使用
> **前置**：🧠 01-core 主控框架
> **核心变更**：所有质检集中到 Phase 5 一次性完成，不打断 Phase 4 执行流

---

## 什么时候加载此技能

- 任务对输出质量有明确要求
- 需要追踪改进趋势
- 需要生成质量报告

---

## 🔍 Phase 0 补充：检查质量系统

执行 Core Phase 0 时，同步检查历史改进记录。

---

## 🛠️ Phase 4 补充说明

> ⚠️ **本插件在 Phase 4 阶段不执行任何操作。**
>
> Core v5.4 规定 Phase 4 为纯执行阶段，所有质检集中到 Phase 5 处理。
> 这样可以保持执行心流，避免 Token 浪费。

---

## 🧹 Phase 5 补充：质量总评

在 Core Phase 5 的"一次性总评与教训总结"环节中，补充以下内容：

### 质量总评模板

```markdown
## Quality 质量总评

### 整体评价
- 完成度：[全部完成 / 部分完成 / 未完成]
- 质量评分：[⭐⭐⭐⭐⭐]
- 评价：[一句话总结]

### 关键经验教训
1. [教训 1]
2. [教训 2]

### 改进建议
- [建议 1]
- [建议 2]

### 历史趋势
- 本次评分：[N]
- 趋势：[📈上升 / 📉下降 / ➡️稳定]
```

### 写入改进日志

将总评追加到 `improvement_log.md`：

```markdown
## [日期] — [项目名称]

**整体质量评分**：⭐⭐⭐⭐
**经验教训**：[总结]
**改进建议**：[建议]
```

---

## 质量评分标准

| 评分 | 含义 | 描述 |
|:----:|------|------|
| ⭐⭐⭐⭐⭐ | 卓越 | 超出预期，可直接交付 |
| ⭐⭐⭐⭐ | 良好 | 满足要求，需微小调整 |
| ⭐⭐⭐ | 及格 | 功能可用，有明显改进空间 |
| ⭐⭐ | 不足 | 未满足要求，需重新执行 |
| ⭐ | 不合格 | 完全偏离目标 |
