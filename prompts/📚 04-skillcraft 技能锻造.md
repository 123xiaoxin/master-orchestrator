## 什么时候加载此技能

- 在执行过程中发现了**可复用的解决路径**
- 经过试错才找到正确方法
- 解决了复杂的多步骤问题
- 想让 Agent 越用越强、不重复踩坑

## 🧹 Phase 4 补充：技能审计

在 Core Phase 4 的"一次性总评与教训总结"环节中，回顾整个任务过程，评估哪些经验值得沉淀：

### 评估标准

| 条件 | 是否沉淀 |
|------|---------|
| 解决路径非平凡（经过试错） | ✅ 是 |
| 可复用于类似场景 | ✅ 是 |
| 有明确的步骤流程 | ✅ 是 |
| 补充了已有 skill 的空白 | ✅ 更新已有 skill |
| 简单的一步操作 | ❌ 否 |
| 纯个人信息（属于 MEMORY.md） | ❌ 否 |
| 已有相同内容的 skill | ❌ 否 |

### 创建新 skill 的格式

```markdown
---
name: skill-name
description: 简短描述
tags: [标签1, 标签2]
---

# Skill 名称

## 场景
[什么情况下使用这个 skill]

## 步骤
1. [步骤 1]
2. [步骤 2]

## 注意事项
- [注意点]
```

### 审计输出模板

```markdown
## Skillcraft 技能沉淀

### 本次可沉淀内容
| 经验 | 类型 | 说明 |
|------|------|------|
| [经验名] | 🆕 新增 / 🔄 更新 | [说明] |

### 跳过的评估
- [N] 次评估被跳过（暂无价值）

### 总结
本次任务共沉淀 [N] 个技能/知识。
```

---

## Skill 质量准则

一个好的 skill 应该：
- ✅ **可操作** — 包含具体步骤，不只是概念
- ✅ **可复用** — 适用于多个相似场景
- ✅ **聚焦** — 一个 skill 解决一类问题
- ✅ **验证过** — 基于实际经验，不是推测
